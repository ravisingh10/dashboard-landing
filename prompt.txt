# dashboard-landing
Dashboard Landing page for Home Server, will not have a simple few landing page


This is a nodejs project for a dashboard which is hosted on the base page
It uses PORT 3001
This project is intended to be used on a localnetwork that contains different links on the landing page

Each card is called a service link card(link to a different service)

Landing Page
This Page is default page when we reach the project
THis page is a page containing header : Links to services on this network
Below that there is a deck of cards 3 in a row scrollable below
show max 10 rows by default and add paginiation for next page in the top and bottom of the deck

There is a search box in the page which searches among the name and description of the service

There is an admin side which uses admin username and password from the User Model
This admin page lets user add and update the existing service link cards
These service link cards are stored in database and the same is rendered in the first page


<!-- Pages -->

Landing Page
    Features:
    1. Paginated display of service link card
    2. Search feature - one field, seaches among title and description
    3. the link cards are rendered from database table

Admin Login Page
    Login page for admin to accept username and password
    Session management - Done by cookies with a signed jwt token
    Add a session middleware that checks the session from cookie - the value in the cokkie is a jwt token that is signed by a secret from .env file
    if the login is success a jwt token is saved in the cokkie containing username, and expiry of 24hrs for the jwt token 

Admin Settings Settings Page:
    Contains cards for different settings pages
    1. Create New Service Link
        This is a page that contains a create form for the model Service Link
        accessible only to logged in user
    2. Update Service Links Listing
        Page to search Service Links
        Pre populated  with a table of search results or default first 20 service Links
        There is a pagination on this page for 20 records per page by default
        accessible only to logged in user
    3. Update Service Link Page
        This page is used to update any existing Service Link
        The page is a form pre-populated with existing value of service link that is to be edited
        The page accepts a path parameter serviceLinkId which is pk for Service link to be edited
        On submit of this page the Service Link is updated 
        accessible only to logged in user


    4. Create User Page
        Page to search Users
        Pre populated  with a table of search results or default last 20 User Model
        There is a pagination on this page for 20 records per page by default
        accessible only to logged in user

    5 Update User Page
        This page is used to update any existing user
        The page is a form pre-populated with existing value of user that is to be edited
        The page accepts a path parameter userId which is pk for user to be edited
        On submit of this page the user is updated 
        accessible only to logged in user


    
    

Models:

User
    password - string hashed text 
    firstName - string required
    lastName - string optional
    userName string not empty
    active Boolean default true
    id - Integer pk
    createdAt - automatically set when record was created
    updated at - automatically set when record was last updated


Serivce Link
    id - pk 
    active Boolean default true
    title String required
    Description String optional
    url- String required
    createdAt - automatically set when record was created
    updated at - automatically set when record was last updated


Libraries:

ORM - sequelize 
database - sqlite
page render engine - ejs
server - express
